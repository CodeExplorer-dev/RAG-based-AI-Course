# API 接口文档

> 项目：基于RAG技术的AI课程答疑与知识点图谱系统
> 基础URL：`http://localhost:5000/api`
> 字符集：UTF-8（请求/响应均使用 JSON 格式）

---

## 通用说明

### 认证方式

除注册和登录外，所有接口需在请求头携带 JWT Token：

```
Authorization: Bearer <access_token>
```

### 通用响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { }
}
```

### 错误响应格式

```json
{
  "code": 400,
  "message": "用户名已存在",
  "data": null
}
```

### 状态码说明

| code | 说明 |
|------|------|
| 200 | 请求成功 |
| 400 | 参数错误 |
| 401 | 未登录 / Token 过期 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 409 | 冲突（如已加入课程、邮箱已被使用） |
| 500 | 服务器内部错误 |

## 一、认证模块 `/api/auth`

### 1.1 用户注册

```
POST /api/auth/register
```

| 项 | 内容 |
|------|------|
| 说明 | 注册新用户账号 |
| 认证 | 否 |

**请求体：**

```json
{
  "username": "zhangsan",
  "password": "123456",
  "email": "zhangsan@example.com",
  "role": "student"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，唯一，3-50字符 |
| password | string | 是 | 密码，6-100字符 |
| email | string | 否 | 邮箱，唯一 |
| role | string | 否 | `student`（默认）/ `teacher` / `admin` |

**成功响应：**

```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "zhangsan",
    "email": "zhangsan@example.com",
    "role": "student",
    "created_at": "2026-06-17 10:30:00"
  }
}
```

---

### 1.2 用户登录

```
POST /api/auth/login
```

| 项 | 内容 |
|------|------|
| 说明 | 登录并获取 Token |
| 认证 | 否 |

**请求体：**

```json
{
  "username": "zhangsan",
  "password": "123456"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**成功响应：**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": 1,
      "username": "zhangsan",
      "email": "zhangsan@example.com",
      "role": "student"

  }
}
```

| 字段 | 说明 |
|------|------|
| access_token | 访问令牌，默认有效期由 JWT 配置决定 |
| refresh_token | 刷新令牌，用于获取新的 access_token |

## 五、AI 问答模块 `/api/chat`

### 5.1 AI 提问（RAG 检索 + LLM 生成）

```
POST /api/chat
```

| 项 | 内容 |
|------|------|
| 说明 | 向 AI 提问，系统通过关键词检索课件分块，结合 LLM 生成回答 |
| 认证 | 是 |

**请求体：**

```json
{
  "message": "什么是过拟合？",
  "course_id": 1
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 是 | 用户问题 |
| course_id | int | 否 | 课程ID，用于限定课件检索范围。不传则全局检索 |

**处理流程：**

```
用户问题 → 关键词拆分 → SQL LIKE 检索 document_chunks（按命中次数排序）
    → 构造 Prompt（系统提示 + 检索片段 + 用户问题） → LLM 生成回答 → 返回结果
```

> 注：当前使用关键词检索作为 RAG 检索方式，后续可升级为 ChromaDB 向量检索。LLM 调用失败时自动降级返回课件原文摘要。

**成功响应（LLM 正常）：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "answer": "**过拟合（Overfitting）**是机器学习中常见的问题...\n\n### 解决方法：\n1. **正则化**：L1/L2正则化...\n2. **Dropout**：训练时随机丢弃神经元...",
    "sources": [
      "片段1: 过拟合（overfitting）是指模型在训练数据上表现得过于完美...",
      "片段2: 防止过拟合的方法包括正则化、Dropout、早停..."
    ],
    "chunk_count": 2
  }
}
```

**成功响应（未找到相关内容）：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "answer": "抱歉，没有在课件中找到与您问题相关的内容。请尝试换个问法，或联系老师上传更多课件。",
    "sources": [],
    "chunk_count": 0
  }
}
```

**成功响应（LLM 降级）：**

```json
{
  "code": 200,
  "message": "LLM 暂不可用，以下为课件原文摘要",
  "data": {
    "answer": "（AI 助手暂不可用，以下是课件中与您问题相关的内容摘要）\n\n[资料1]\n过拟合（overfitting）是指模型...",
    "sources": [],
    "chunk_count": 3
  }
}
```

## 八、LLM 环境变量配置

AI 问答功能（`/api/chat`）依赖 LLM 服务，需在 `back-end/.env` 中配置：

```env
# LLM API 配置（兼容 OpenAI API 格式）
LLM_API_BASE=https://api.openai.com/v1
LLM_API_KEY=sk-your-api-key-here
LLM_MODEL=gpt-3.5-turbo

# 可选参数
LLM_MAX_TOKENS=1024
LLM_TEMPERATURE=0.7
```

未配置时，AI 问答自动降级为课件原文摘要检索。

# API 接口文档

> 项目：基于RAG技术的AI课程答疑与知识点图谱系统
> 基础URL：`http://localhost:5000/api`
> 字符集：UTF-8（请求/响应均使用 JSON 格式）

---

## 通用说明

### 认证方式

除注册和登录外，所有接口需在请求头携带 JWT Token：

```
Authorization: Bearer <access_token>
```

### 通用响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { }
}
```

### 错误响应格式

```json
{
  "code": 400,
  "message": "用户名已存在",
  "data": null
}
```

### 状态码说明

| code | 说明 |
|------|------|
| 200 | 请求成功 |
| 400 | 参数错误 |
| 401 | 未登录 / Token 过期 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 一、认证模块 `/api/auth`

### 1.1 用户注册

```
POST /api/auth/register
```

| 项 | 内容 |
|------|------|
| 说明 | 注册新用户账号 |
| 认证 | 否 |

**请求体：**

```json
{
  "username": "zhangsan",
  "password": "123456",
  "email": "zhangsan@example.com",
  "role": "student"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，唯一，3-50字符 |
| password | string | 是 | 密码，6-100字符 |
| email | string | 否 | 邮箱，唯一 |
| role | string | 是 | `student` / `teacher` / `admin` |

**成功响应：**

```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "zhangsan",
    "email": "zhangsan@example.com",
    "role": "student",
    "created_at": "2026-06-17 10:30:00"
  }
}
```


| 项 | 内容 |
|------|------|

| 认证 | 否 |

**请求体：**

```json

```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

**成功响应：**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {

    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
```

## 四、课件模块 `/api/courseware`

### 4.1 上传课件

```
POST /api/courseware/upload
```

| 项 | 内容 |
|------|------|
| 说明 | 上传课件文档（PDF/PPT/DOCX等），上传后系统自动解析、分块、向量化 |
| 认证 | 是 |
| 权限 | 仅 `teacher` 和 `admin` 角色 |
| Content-Type | multipart/form-data |

**请求体（form-data）：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 课件文件，最大 50MB |
| course_id | int | 是 | 所属课程ID |
| title | string | 否 | 课件标题，不传则使用文件名 |

**支持的文件格式：**

| 格式 | 扩展名 |
|------|--------|
| PDF | `.pdf` |
| PowerPoint | `.ppt` `.pptx` |
| Word | `.doc` `.docx` |
| 纯文本 | `.txt` |
| Markdown | `.md` |

**成功响应：**

```json
{
  "code": 200,
  "message": "上传成功，正在解析中",
  "data": {
    "id": 1,
    "title": "第一章 绪论",
    "file_type": "pdf",
    "file_size": 2048000,
    "course_id": 1,
    "status": "processing",
    "uploaded_at": "2026-06-17 14:00:00"
  }
}
```

> 课件上传后立即返回 `status: "processing"`。后端异步进行文档解析、分块和向量化，完成后自动更新 `status` 为 `"completed"`。

### 5.4 RAG 提问（核心接口）

```
POST /api/qa/ask
```

| 项 | 内容 |
|------|------|
| 说明 | 向 AI 提问，系统自动进行 RAG 检索并生成回答 |
| 认证 | 是 |

**请求体：**

```json
{
  "conversation_id": 1,
  "course_id": 1,
  "question": "什么是过拟合？有哪些解决方法？"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| conversation_id | int | 否 | 会话ID，不传则自动创建新会话 |
| course_id | int | 否 | 课程ID，用于限定检索范围 |
| question | string | 是 | 用户问题，1-2000字符 |

**处理流程（后端自动完成）：**

```
用户问题 → Embedding向量化 → ChromaDB语义检索Top-5分块 → 
构造Prompt（参考资料+问题）→ LLM生成回答 → 记录到数据库 → 返回结果
```

**成功响应：**

### 6.2 触发知识点提取

```
POST /api/knowledge/extract
```

| 项 | 内容 |
|------|------|
| 说明 | 触发 LLM 自动从课件中提取知识点和关系，构建知识图谱 |
| 认证 | 是 |
| 权限 | 仅该课程的授课教师 |

**请求体：**

```json
{
  "course_id": 1
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| course_id | int | 是 | 课程ID |

**成功响应：**

```json
{
  "code": 200,
  "message": "知识点提取已触发，正在处理中",
  "data": {
    "course_id": 1,
    "status": "processing",
    "estimated_time": "约需 30-60 秒"
  }
}
```

> 知识点提取是异步操作，由 LLM 分析课件内容后自动写入 `knowledge_points` 和 `knowledge_point_relations` 表。提取完成后可通过 `GET /api/knowledge/graph` 获取最新图谱数据。

## 七、反馈模块 `/api/feedback`

### 7.1 提交反馈

```
POST /api/feedback
```

| 项 | 内容 |
|------|------|
| 说明 | 对某条 AI 回答进行评价反馈 |
| 认证 | 是 |

**请求体：**

```json
{
  "message_id": 2,
  "rating": 4,
  "feedback_type": "helpful",
  "comment": "回答很详细，引用来源清晰"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message_id | int | 是 | 被评价的 AI 消息ID |
| rating | int | 是 | 评分 1-5 |
| feedback_type | string | 否 | `helpful` / `not_helpful` / `inaccurate` / `incomplete` / `other` |
| comment | string | 否 | 文字反馈 |

**成功响应：**

```json
{
  "code": 200,
  "message": "反馈提交成功",

    "id": 1,
    "user_id": 1,
    "message_id": 2,
```
